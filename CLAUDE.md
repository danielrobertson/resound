## Product

**Resound** ‚Äî an app for student musicians to record music lessons (video, audio clips, notes) and recall any moment fast via name, tag, time, and AI-extracted related topics. Tagline: *lessons that continue to resonate.*

## Stack

TanStack Start (React 19, SSR) deployed to **Cloudflare Workers** via the `@cloudflare/vite-plugin`. Tailwind v4, ShadCn UI, Better Auth (email/password) on Drizzle + Cloudflare D1. Package manager is **pnpm** (pinned via `packageManager`; the README's `npm` instructions are stale ‚Äî use pnpm).

## Commands

```bash
pnpm dev                 # dev server on :3000 (Workers runtime via vite plugin)
pnpm build               # production build (client + SSR worker bundle)
pnpm typecheck           # tsc --noEmit ‚Äî run this to verify changes
pnpm test                # vitest run (all)
pnpm exec vitest run path/to/file.test.ts     # single test file
pnpm exec vitest run -t "test name"           # single test by name

pnpm cf-typegen          # regenerate worker-configuration.d.ts from wrangler.jsonc
pnpm db:generate         # drizzle-kit generate ‚Äî emit SQL migration from schema
pnpm db:migrate:local    # apply migrations to LOCAL D1 (.wrangler/state)
pnpm db:migrate:remote   # apply migrations to REMOTE D1
pnpm deploy              # build + wrangler deploy
```

Vitest uses `@cloudflare/vitest-pool-workers` (see `vitest.config.mts`), so tests run inside the Workers runtime.

**Verification: `pnpm typecheck` passing is sufficient.** Do not spin up the dev server or otherwise confirm live behavior after a change unless the user explicitly asks ‚Äî a clean typecheck is the accepted bar for done.

## Architecture

### Cloudflare bindings (the central pattern)
Bindings are accessed at **module scope** via `import { env } from "cloudflare:workers"` ‚Äî see `src/lib/db/index.ts` (`drizzle(env.DB)`) and `src/lib/auth.ts`. This works because the vite plugin resolves the binding from request context. Binding/secret **types** come from two files:
- `worker-configuration.d.ts` ‚Äî generated by `pnpm cf-typegen` from `wrangler.jsonc`. **Never edit by hand**; rerun cf-typegen after changing bindings.
- `src/env.d.ts` ‚Äî hand-maintained augmentation of `Cloudflare.Env` for `.dev.vars` secrets (`BETTER_AUTH_SECRET`, `BETTER_AUTH_URL`) that don't appear in `wrangler.jsonc`.

### Auth (Better Auth)
- `src/lib/auth.ts` ‚Äî server instance: Drizzle adapter (`provider: "sqlite"`) over D1, email/password, `tanstackStartCookies()` plugin (must stay last in the plugins array).
- `src/routes/api/auth/$.ts` ‚Äî catch-all server route delegating every `/api/auth/*` request to `auth.handler`.
- `src/lib/auth-client.ts` ‚Äî browser client; exports `signIn`, `signUp`, `signOut`, `useSession`.
- Better Auth enforces CSRF: non-GET requests need an `Origin` header matching `BETTER_AUTH_URL`.

### Database & migrations
- `src/lib/db/schema.ts` is **hand-maintained** to match Better Auth's expected tables (user/session/account/verification). It is deliberately *not* generated by the Better Auth CLI, because the CLI can't import `cloudflare:workers` in plain Node.
- Workflow after any schema change: `pnpm db:generate` ‚Üí `pnpm db:migrate:local` (and `:remote` before deploying). `wrangler.jsonc` points `migrations_dir` at `drizzle/`.
- **Local and remote D1 are separate databases.** `db:migrate:local` writes to `.wrangler/state`; the deployed Worker uses the remote `resound-db`. Data does *not* sync between them ‚Äî a user created on `localhost` does not exist in prod. **`wrangler deploy` ships code only; it never runs migrations.** So a fresh schema change (or a first deploy) requires `pnpm db:migrate:remote` explicitly. Symptom of skipping it: the app deploys fine but every request 500s with `Failed query: select ... from "user"` (missing table) ‚Äî check live logs via `wrangler tail resound`.

### Media storage (R2)
- Recording bytes (uploaded files, and later camera/mic captures) live in the **R2 bucket bound as `env.MEDIA`** (`resound-media` in `wrangler.jsonc`); the queryable metadata is the `recording` table in D1. Upload flow: `src/routes/api/recordings.ts` (POST, auth-checked, `env.MEDIA.put` + insert row) ‚Üê client `fetch('/api/recordings', FormData)` in `routes/studio.tsx`; `src/lib/recordings.ts#listRecordings` reads them back for the `/studio` loader (the signed-in app route).
- **Local R2 is simulated in `.wrangler/state`** by the vite plugin/miniflare, so uploads work in `pnpm dev` with no remote bucket. **Before deploying you must create the remote bucket once:** `wrangler r2 bucket create resound-media` (uses the account from `wrangler.jsonc` ‚Äî do not export `CLOUDFLARE_ACCOUNT_ID`). Like D1, local and remote buckets are separate and never sync.

### Routing & document shell
- File-based routes in `src/routes/`. `src/routeTree.gen.ts` is auto-generated ‚Äî **do not edit**; it regenerates on dev/build (or `pnpm generate-routes`).
- `src/routes/__root.tsx` is the HTML shell (renders `Header`/`Footer` around the `Outlet`). It suppresses the site chrome for full-bleed auth pages via `CHROMELESS_ROUTES` (`/login`, `/signup`). An inline script there sets the theme class before hydration.

### Theming & UI conventions
- **ShadCn "radix-nova" style, neutral-gray base with a green primary.** All color comes from semantic tokens in `src/styles.css` (`bg-background`, `text-muted-foreground`, `border-border`, `bg-primary`, ‚Ä¶) defined as OKLCH vars in `:root`/`.dark`. The palette was applied via a shadcn theme preset (`shadcn apply --preset`), which also imports `shadcn/tailwind.css` and adds the `--radius-2xl/3xl/4xl` and `--font-heading` tokens. Do not reintroduce the old TanStack teal/`--sea-ink`/`island-shell` tokens ‚Äî they were removed.
- Dark mode uses the **`.dark` class** (toggled by `ThemeToggle` + the root inline script), wired via `@custom-variant dark`. Inter is the sans font (`--font-sans`), self-hosted via `@fontsource-variable/inter` and applied globally by `html { @apply font-sans }`. Display headings use **Bricolage Grotesque** (`--font-heading`, self-hosted via `@fontsource-variable/bricolage-grotesque`) applied per-element with the `font-heading` utility ‚Äî body copy stays Inter.
- **Icons: Phosphor only** ‚Äî `import { X } from "@phosphor-icons/react/dist/ssr"` with `weight="light"`. `lucide-react` is uninstalled; do not add it back.
- Adding ShadCn components: `pnpm dlx shadcn@latest add <component>`. `components.json` targets `src/styles.css` and `iconLibrary: "phosphor"`, but the CLI may still emit lucide-named icon imports ‚Äî swap them to Phosphor by hand.

## Deployment notes

- Two Cloudflare accounts are visible to this login; `account_id` is set in `wrangler.jsonc` (the `danielrobertson` account). Run `wrangler`/`pnpm db:migrate:remote` with the OAuth login's **default** account ‚Äî do *not* export `CLOUDFLARE_ACCOUNT_ID`, which overrides it and yields `7403 ‚Äì account is not authorized`. (Deploy and `wrangler tail` already resolve the account from `wrangler.jsonc`.)
- D1 database `resound-db` is bound as `env.DB`.
- Before first deploy: `pnpm db:migrate:remote`, then `wrangler secret put BETTER_AUTH_SECRET`, and set `BETTER_AUTH_URL` to the deployed origin.
- **Durable Objects are intentionally not set up yet** ‚Äî add when a stateful-coordination need is confirmed.
